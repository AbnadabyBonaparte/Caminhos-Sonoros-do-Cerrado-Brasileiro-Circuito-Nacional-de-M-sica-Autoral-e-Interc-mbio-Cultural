## 3. 🎯 A TESE WHITE-LABEL — e ela é melhor do que parece

### O que o Fundador afirmou
> *"Esse aqui é outro white-label."*

### ✅ A tese procede — mas por um motivo diferente do jogo

**Tecnicamente**, hoje **não** é white-label: o conteúdo está **embutido no HTML** (494 linhas de `index.html`), não em arquivo de dados.

**Comercialmente**, porém, a tese é forte — e o mercado é maior do que o repositório sugere.

### O mercado real: projeto cultural financiado no Brasil
Todo ano, milhares de projetos são aprovados em:
**Funarte** · **Lei Rouanet** · **Lei Paulo Gustavo** · **Lei Aldir Blanc** · editais municipais e estaduais

**E praticamente todos precisam da mesma coisa:**

| Necessidade | Por quê |
|---|---|
| Página institucional | **frequentemente exigida no edital** |
| Formulário de inscrição | oficinas, seleção de participantes |
| Programação e cronograma | divulgação das etapas |
| Galeria de fotos | **prestação de contas** |
| Logos de financiador e apoiadores | **obrigação legal do edital** |
| Contato institucional | transparência |

👉 **É exatamente o que este repositório já entrega.** As nove seções construídas — hero, sobre, circuito, atividades, quem realiza, galeria, formulário, contato, rodapé institucional — **são o esqueleto padrão de qualquer projeto cultural financiado.**

### 💰 Por que este cliente é bom
1. **O dinheiro já existe** — o projeto foi aprovado e tem verba. Não se vende para quem *talvez* tenha orçamento.
2. **O site costuma ser item do edital** — não é gasto opcional; é obrigação de prestação de contas.
3. **Ciclo curto e escopo definido** — não há negociação longa; há prazo de execução.
4. **Recorrência natural** — o mesmo produtor aprova projeto novo todo ano.
5. **Setor por indicação** — um cliente satisfeito abre três.

## 7. ▶️ O CAMINHO — transformar entrega em produto

**1. Extrair o conteúdo para `projeto.json`** *(horas)*
Nome, descrição, etapas, atividades, realizador, financiadores, logos, contatos. **Depois disso, projeto novo = arquivo novo.**

**2. Gravar as inscrições em banco** *(pequeno, mas importante)*
Edital exige prestação de contas de participantes. E-mail não é registro auditável. Usar o Banco do Universo com `mundo='alsham'`.

**3. Montar o "kit edital"** — um produto nomeado:
> *"Site institucional para projeto cultural aprovado — pronto em X dias, com acessibilidade WCAG, formulário de inscrição, galeria de prestação de contas e logos dos financiadores conforme manual do edital."*

**4. Usar a relação existente como canal** — a realizadora recorrente conhece outros produtores. É a porta mais barata que a ALSHAM tem para um mercado inteiro.

**5. Renomear o repositório.**
